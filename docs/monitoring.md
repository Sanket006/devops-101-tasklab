# Monitoring Guide (Prometheus & Grafana)

This guide covers deploying the Prometheus and Grafana monitoring stack onto your Kubernetes cluster and configuring metric collection for the Task Manager application.

## 1. Deploying the Monitoring Stack via Helm

We use the standard community-supported `kube-prometheus-stack` chart, which installs Prometheus Operator, Prometheus server, Node Exporter, kube-state-metrics, and Grafana.

1. **Add Helm Repository and update:**
   ```bash
   helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
   helm repo update
   ```

2. **Install the Chart:**
   Install the stack into the dedicated `monitoring` namespace:
   ```bash
   helm install prometheus prometheus-community/kube-prometheus-stack \
     --namespace monitoring \
     --create-namespace
   ```

3. **Verify the Installation:**
   Check that all components are running correctly (this may take a couple of minutes to pull all images):
   ```bash
   kubectl get pods -n monitoring
   ```

## 2. Metric Scraping Configuration

The Flask application exposes standard Prometheus-compatible metrics on the `/metrics` endpoint (powered by the `prometheus-client` Python SDK).

Prometheus auto-discovers and scrapes metrics from any pod carrying specific annotations. Our application pod template in [deployment.yaml](file:///c:/Users/Lenovo/Sanket%20Personal/simple-devops-project/k8s/deployment.yaml#L61-L64) is pre-configured with these annotations:

```yaml
spec:
  template:
    metadata:
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/path: "/metrics"
        prometheus.io/port: "5000"
```

No additional Custom Resource Definitions (CRDs) like `ServiceMonitor` are needed for basic scraping in a standard configuration.

## 3. Querying Metrics in Prometheus

To access the Prometheus dashboard and run queries:

1. **Port-forward the Prometheus server:**
   ```bash
   kubectl port-forward svc/prometheus-kube-prometheus-prometheus -n monitoring 9090:9090
   ```

2. **Access the Web Console:**
   Open your browser to: **[http://localhost:9090](http://localhost:9090)**

3. **Search for Application Metrics:**
   Try running these custom metrics generated by our Flask app:
   * `tasks_total` — Total tasks created.
   * `tasks_done_total` — Total tasks marked completed.
   * `tasks_pending_total` — Total tasks currently pending.
   * `flask_http_request_total` — Total requests served by Flask.

## 4. Visualizing Metrics in Grafana

To design dashboards and visualize metrics:

1. **Port-forward the Grafana service:**
   ```bash
   kubectl port-forward svc/prometheus-grafana -n monitoring 3000:80
   ```

2. **Access the Web UI:**
   Open your browser and navigate to: **[http://localhost:3000](http://localhost:3000)**

3. **Log in:**
   * **Username:** `admin`
   * **Password:** Retrieve the auto-generated password using one of these commands:
     * **On PowerShell:**
       ```powershell
       [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String((kubectl get secret --namespace monitoring prometheus-grafana -o jsonpath="{.data.admin-password}")))
       ```
     * **On Bash/Zsh:**
       ```bash
       kubectl get secret --namespace monitoring prometheus-grafana -o jsonpath="{.data.admin-password}" | base64 --decode ; echo
       ```

---

## 5. Cleaning Up

To delete the monitoring stack and free up system resources:
```bash
helm uninstall prometheus -n monitoring
kubectl delete namespace monitoring
```
